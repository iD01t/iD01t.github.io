Introduction to HTML2EXE Professional v28.0

Welcome to the definitive guide for HTML2EXE Professional v28.0, the ultimate tool for transforming HTML-based web applications into standalone desktop executables (.exe for Windows, .app for macOS, and binaries for Linux). Built by iD01t Productions, this production-release software combines a intuitive CustomTkinter GUI with a robust CLI, powered by PyInstaller and pywebview, to deliver flawless builds every time.

Key Features:

  • GUI Simplicity: Drag-and-drop configuration for source folders/URLs, window settings, and build options.
  • CLI Power: Headless mode for automation and CI/CD pipelines.
  • License Compliance Scanner: Scans dependencies for GPL/MIT/Apache compliance to ensure commercial safety.
  • Cross-Platform: Builds for Windows, macOS, and Linux from any platform.
  • Pro Enhancements: Icon support, single-file bundling, progress tracking, and logging.

This guide is your blueprint: From zero-setup installation to advanced troubleshooting, we've covered every angle to save you dev hours. Whether you're an indie hacker shipping MVPs or a senior dev scaling agency workflows, HTML2EXE Pro is your flawless companion.

Installation and Setup

Prerequisites

HTML2EXE requires Python 3.8+ (tested up to 3.13). No admin rights needed for user installs.

PlatformRequirements
WindowsPython 3.8+, Visual C++ Build Tools (for pywebview compilation)
macOSPython 3.8+, Xcode Command Line Tools
LinuxPython 3.8+, GTK3 (for webview)

Step-by-Step Installation

  1. Install Python: Download from python.org. Check "Add to PATH" during setup.
  2. Create Virtual Environment (Recommended): 
    python -m venv html2exe_venv
                        # On Windows: html2exe_venv\Scripts\activate.bat
                        # On macOS/Linux: source html2exe_venv/bin/activate
  3. Install Dependencies: 
    pip install --upgrade pip
                        pip install customtkinter pywebview pyinstaller Pillow requests

    Note for Windows: If pywebview fails, install Microsoft C++ Build Tools from here, then retry.

  4. Download & Run:
    • GUI: python html2exe.py or double-click HTML2exe.exe.
    • CLI: python html2exe.py check-deps to verify.
  5. Build the Standalone .exe (Optional): 
    pyinstaller --name HTML2exe --onefile --windowed --icon=assets/icon.ico html2exe.py

Logs Location: %USERPROFILE%\.html2exe_pro\app.log (Windows) or ~/.html2exe_pro/app.log (others).

Pro Tip: Use --user flag outside venv to avoid permission issues: python -m pip install --user ....

GUI Walkthrough: Building Your First App

The GUI is a dark-themed, responsive interface built with CustomTkinter. Launch it via python html2exe.py or HTML2exe.exe. It features a sidebar for navigation and tabs for configuration.

Project Tab: Core Configuration

This tab sets up your app's metadata and source content.

  1. App Name: Enter your executable's name (e.g., "MyDashboard"). Validates for alphanumeric chars.
  2. Version: Semantic versioning (e.g., "1.0.0"). Ensures format compliance.
  3. Author: Your name/company for metadata.
  4. Source Content:
    • Select Local Folder: Browse to a directory with index.html (e.g., your React/Vue build output).
    • Select URL: Enter a full HTTPS URL (e.g., "https://example.com/app"). Validates regex.

Validation: Real-time checks prevent invalid paths; tooltips guide inputs.

Window Tab: User Experience Settings

Customize the desktop window for your end-users.

  1. Title: Window bar text (e.g., "My Secure Dashboard").
  2. Initial Size: Width x Height in pixels (default: 1024x768). Responsive tip: Test on various screens.
  3. Resizable Window: Checkbox for dynamic sizing.
  4. Start in Fullscreen: Checkbox for immersive mode.
Pro Tip: For kiosk apps, enable fullscreen and disable resizable for locked-down UX.

Build Tab: Output Options

Define how your executable is packaged.

  1. Single File Executable: Checkbox for --onefile (default: on). Slower startup but portable.
  2. Show Console Window: Checkbox for debugging (--console). Disable for production.
  3. Application Icon: Browse for .ico (Windows) or .icns (macOS). Validates file type per platform.
  4. Output Dir: Folder for artifacts (default: "dist"). Creates if missing.

Build Process: Click BUILD (green button). Progress bar tracks steps: Validate (5%), Launcher (15%), Resources (25%), PyInstaller (35%+). Cancel anytime.

Licenses Tab: Compliance Check

Scan for dependency licenses to avoid GPL pitfalls in commercial distro.

  1. Click Scan Now. Outputs table: Package, Version, License, Status (ALLOWED/RESTRICTED/WARNING).
  2. Example: PyInstaller v6.16.0 | GPL-2.0 with exceptions | ALLOWED
  3. Export: Copy-paste for your LICENSES.txt.

Rules: MIT/BSD/Apache = ALLOWED; GPL/AGPL = RESTRICTED.

CLI Reference: Automation Mastery

For scripts, CI/CD (GitHub Actions/Jenkins), use python html2exe.py <command> or HTML2exe.exe <command>. Headless mode auto-detects.

Commands

CommandDescriptionExample
check-depsVerify dependenciespython html2exe.py check-deps
licenses [--format table|json]Scan licensespython html2exe.py licenses --format json
build --source <path/url> [--project <json>] [--name <name>] [--output <dir>] [--console]Build executablepython html2exe.py build --source ./myapp --name MyApp --output dist

Build Flags Deep-Dive

  • --source: Required; folder with index.html or URL.
  • --project: JSON config file overrides flags.
  • --name: Executable name (default: HTMLApp).
  • --output: Dist folder (default: dist).
  • --console: Enables debug console.

Output: Progress logs to stdout; artifact path on success.

Pro Tip: Integrate with GitHub Actions: Use build command in a matrix for cross-platform testing.

Advanced Features: Pro-Level Customization

Project Files (.json)

Save/Load configs for teams. Structure:

{
  "app_name": "MyApp",
  "app_version": "1.0.0",
  "source_type": "folder",
  "source_path": "./src",
  "window_width": 1024,
  "single_file": true
}

Load via GUI (Save/Load buttons) or CLI (--project).

Custom Icons & Resources

Icons: .ico (16x16 to 256x256 PNG converted). Resources auto-bundle from source folder.

Headless Mode

Run with args like "build" to skip GUI. Ideal for servers.

Logging & Progress

Rotating logs (2MB max, 5 backups). GUI console mirrors file logs.

Pro Tip: Hook into webview events for custom JS bridges in advanced apps.

Troubleshooting: Bulletproof Fixes

99% of issues stem from deps or paths. Always check logs first.

IssueSymptomSolution
Missing Dependencies"PyInstaller not found"pip install pyinstaller customtkinter pywebview
Build Fails: No index.htmlValidation errorEnsure source folder has index.html or valid URL
Icon Error"Invalid .ico"Use online converter (e.g., icoconvert.com); match platform
pywebview Compile Fail (Windows)C++ 14.0 requiredInstall Visual Studio Build Tools
Large .exe Size>100MBUse --onedir; compress with UPX: pyinstaller --upx-dir /path/to/upx
Antivirus FlagsFalse positiveSign with EV cert (e.g., DigiCert); whitelist in testing
Cross-Platform IssuesmacOS/Linux failBuild on target OS; add --hidden-import=webview.platforms.[platform]

Debug Mode

Enable --console, tail logs: tail -f ~/.html2exe_pro/app.log.

Pro Tip: For PyInstaller warnings, inspect build/warn-*.txt for hidden imports.

Best Practices: Senior Dev Workflow

  • Test Iteratively: Build small prototypes; validate on target hardware.
  • Compliance First: Scan licenses pre-release; include LICENSES.txt in distro.
  • Optimize Size: Minify HTML/JS/CSS; exclude dev files from source.
  • Security: Use HTTPS URLs; sandbox webview if needed.
  • Version Control: Git-track .json projects; automate builds with CLI in CI.
  • Updates: Poll APP_UPDATE_URL for patches.
Pro Tip: For Electron migrants, HTML2EXE is lighter—bundle only essentials for sub-50MB apps.
